# Local RAG – Hybrid Search Q&A System

A fully local Retrieval-Augmented Generation system that combines **semantic search**
(ChromaDB + sentence-transformers) with **keyword search** (BM25) to answer questions
about a curated set of Wikipedia articles. Answers are generated by a locally running
LLM via Ollama—no data leaves your machine.

## How It Works

The system operates in three stages:

### 1. Data Ingestion (`ingest.py`)

Fetches plain-text content from Wikipedia for 46 entities (21 people + 25 places).
The script tries the English Wikipedia first, and falls back to Turkish Wikipedia if
the article isn't found. Each article is saved as a `.txt` file in the `data/` directory.

### 2. Database Building (`build_db.py`)

Processes the raw text files into two search indices:

- **Semantic Index (ChromaDB)**: Each document is split into overlapping chunks (1000 chars,
  100 char overlap), embedded using the `all-MiniLM-L6-v2` model, and stored in a ChromaDB
  collection with cosine similarity.
- **Keyword Index (BM25)**: The same chunks are tokenized and indexed using BM25Okapi for
  traditional keyword matching. The index is serialized to `bm25_index.pkl`.

### 3. Interactive Q&A (`app.py`)

A Streamlit web interface where you can ask questions in natural language. For each query:

1. **Semantic Search** finds chunks with similar meaning using vector similarity.
2. **BM25 Search** finds chunks with matching keywords.
3. **Reciprocal Rank Fusion (RRF)** merges both result sets into a single ranked list
   (semantic weight: 0.9, BM25 weight: 0.1, k=60).
4. The top-K contexts are passed to **Ollama (llama3.2)** which generates a grounded,
   streaming answer.

## Prerequisites

- **Python 3.10+**
- **Ollama** installed and running locally with the `llama3.2` model pulled:
  ```bash
  ollama pull llama3.2
  ```

## Installation

```bash
# Clone the repository
git clone https://github.com/yourusername/local-rag.git
cd local-rag

# Install Python dependencies
pip install -r requirements.txt
```

## Usage

Run the three stages in order:

```bash
# Step 1: Fetch Wikipedia articles
python ingest.py

# Step 2: Build the search indices
python build_db.py

# Step 3: Launch the Streamlit app
streamlit run app.py
```

The app will open in your browser. Use the chat input to ask questions about any of the
indexed entities. Adjust the Top-K slider in the sidebar to control how many context
chunks are retrieved.

## Tech Stack

| Component         | Technology                          |
|-------------------|-------------------------------------|
| Data Source        | Wikipedia MediaWiki API             |
| Embeddings        | `all-MiniLM-L6-v2` (sentence-transformers) |
| Vector Store      | ChromaDB (persistent, cosine)       |
| Keyword Search    | BM25Okapi (rank_bm25)              |
| Fusion            | Reciprocal Rank Fusion (RRF)        |
| LLM               | Ollama – `llama3.2` (local)         |
| UI                | Streamlit                           |

## Project Structure

```
local-rag/
├── ingest.py            # Fetches Wikipedia articles to data/
├── build_db.py          # Builds ChromaDB + BM25 indices
├── app.py               # Streamlit hybrid search Q&A interface
├── requirements.txt     # Python dependencies
├── data/                # 46 Wikipedia article text files
├── chroma_db/           # ChromaDB persistent storage
└── bm25_index.pkl       # Serialized BM25 index
```

## Configuration

Key parameters can be adjusted in the source files:

| Parameter           | File        | Default          | Description                  |
|---------------------|-------------|------------------|------------------------------|
| `CHUNK_SIZE`        | build_db.py | 1000 chars       | Character size per chunk     |
| `CHUNK_OVERLAP`     | build_db.py | 100 chars        | Overlap between chunks       |
| `SEMANTIC_WEIGHT`   | app.py      | 0.9              | RRF weight for semantic      |
| `BM25_WEIGHT`       | app.py      | 0.1              | RRF weight for BM25          |
| `RRF_K`             | app.py      | 60               | RRF smoothing constant       |
| `OLLAMA_MODEL`      | app.py      | `llama3.2`       | Ollama model name            |
| `EMBEDDING_MODEL`   | build_db.py | `all-MiniLM-L6-v2` | Sentence-transformers model |
